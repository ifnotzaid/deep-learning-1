# Project Lead & Primary Developer

## Overview
I was the main developer on this project. I handled basically everything
from setting up the Colab environment to writing the final report. The
others helped with research, analysis and the presentation but the
implementation and debugging was mostly on me.

## What I Did

### Setup & Infrastructure
- Set up the Google Colab notebook with GPU and built the Google Drive
  save/restore system — this saved us multiple times when Colab
  disconnected mid-training
- Set up the GitHub repo, wrote REPORT.md, and organized the figures folder
- Handled all the PyTorch version issues (the weights_only=False bug with
  PyTorch 2.6 when loading numpy checkpoints took embarrassingly long to fix)

### MLP Implementation (Cells 1–12)
- Implemented the full NumPy MLP from scratch — forward pass, backward pass,
  cross-entropy, SGD with LR decay, all gradients by hand
- Built the universal training engine with early stopping that all models
  share — wanted one function so comparisons would be fair
- Trained all 6 MLP variants and ran the hyperparameter sweep
  (λ sweep and dropout sweep on the validation set only)
- Implemented FGSM adversarial attack and the 5-fold cross-validation
- Debugged several issues along the way including patience being too
  aggressive in early stopping and the learning rate being too low
  in the initial NumPy MLP

### CNN & Optimization (Cells 13–19)
- Designed and implemented the CNN architecture
- Ran the full optimizer shootout and LR scheduling comparison
- The warmup implementation had a bug that kept the LR near zero —
  diagnosed it and fixed it in the same session
- Ran batch size analysis and produced the final comparison charts

### Presentation
- Prepared and delivered the full project presentation covering all
  four weeks of methodology and experimental findings
