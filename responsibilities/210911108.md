# Data, Evaluation & Repository

## Overview
Focused on the parts that are easy to get wrong if you're not careful —
data handling, evaluation discipline, and making sure the repo is clean
for submission.

## Data & Evaluation

The train/val/test split sounds simple but there are a lot of ways to
accidentally leak information. Made sure the validation set was only
ever used for hyperparameter tuning and the test set was genuinely
touched once at the end. Had to remind the group about this a few
times during the hyperparameter tuning phase.

Verified the class distribution — all 10 classes have exactly 5000
examples which satisfies the i.i.d. assumption. Also checked that the
augmentation transforms (horizontal flip, random crop, color jitter)
all preserve the class label, which isn't always obvious. A flipped cat
is still a cat. A flipped 6 would be a 9 — good thing we're not doing
digit recognition.

## K-Fold Cross-Validation

Ran the 5-fold CV and computed the confidence intervals. The tight
standard deviation (0.0034) was actually reassuring — it means the
model is stable and the val accuracy isn't just getting lucky on one
particular split. SE = σ/√m = 0.0015 which is very precise.
